## ??? 点击步骤演示

### 步骤1：首次启用
1. 打开仓库页面
2. 点击 **"Actions"** 标签页
3. 看到欢迎页面
4. 点击 **"I understand my workflows, go ahead and enable them"**

### 步骤2：查看工作流
1. 在左侧菜单，点击 **"All workflows"**
2. 在右侧列表，找到 **"番茄钟项目PR状态报告"**
3. 点击工作流名称进入详情页

### 步骤3：手动触发
1. 在工作流详情页，点击 **"Run workflow"** 按钮
2. 选择分支：**"main"** 或 **"master"**
3. 点击绿色的 **"Run workflow"** 按钮
4. 等待工作流运行完成

### 步骤4：查看结果
1. 在运行列表中，点击刚完成的运行
2. 展开 **"Generate report"** 步骤
3. 查看输出日志
4. 点击 **"Artifacts"** 下载报告

- ? **绿色对勾**：运行成功

## ?? 高级设置

### 工作流权限设置
1. 在仓库页面，点击 **"Settings"**
2. 左侧菜单，点击 **"Actions"** → **"General"**
3. 在 **"Workflow permissions"** 部分：
   - **"Read and write permissions"**：允许读写
   - **"Allow GitHub Actions to create and approve pull requests"**：允许创建PR

### 环境变量设置
1. 在工作流文件中定义：
```yaml
env:
  PROJECT_NAME: "番茄钟"
  REPORT_PATH: "reports/"
```

2. 或者在仓库设置中：
   - Settings → Secrets and variables → Actions
   - 添加新的变量

## ?? 实际操作示例

### 场景1：查看今天的报告
1. 打开Actions页面
2. 点击工作流名称
3. 找到今天的运行（按时间排序）
4. 点击运行查看详情
5. 下载报告文件

### 场景2：手动触发报告生成
1. 在工作流详情页
2. 点击 "Run workflow"
3. 选择 "main" 分支
4. 点击 "Run workflow"
5. 等待1-2分钟完成

### 场景3：暂停自动化
1. 在工作流详情页
2. 点击 "Disable workflow"
3. 确认禁用
4. 工作流将停止自动运行

### 场景4：查看历史趋势
1. 在工作流详情页
2. 查看所有运行列表
3. 观察成功/失败趋势
4. 分析运行时间变化
