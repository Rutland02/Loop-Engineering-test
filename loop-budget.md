# Loop Budget - 番茄钟项目

> 主循环：**PR Babysitter** (用于监控和修复番茄钟项目的PR)

## 每日限制

| 循环模式 | 每天最大运行次数 | 每天最大Token数 | 每次运行最大子代理数 |
|----------|----------------|----------------|-------------------|
| PR Babysitter | 288 | 2M | 0 (L1) / 3 (L2) |
| CI修复 | 50 | 500K | 2 |
| 功能开发 | 20 | 200K | 1 |

## 预算超限处理

1. 暂停调度器（禁用自动修复）
2. 在 `loop-run-log.md` 中记录事件
3. 通知人类（通过Slack/Issue/STATE.md高优先级）

## 紧急停止

- 命令或Issue标签：`loop-pause-all`
- 仅在人类清除STATE.md中的标志后才能恢复

## 成本估算

```bash
# 估算番茄钟项目的运行成本
npx @cobusgreyling/loop-cost --pattern pr-babysitter

# 估算CI修复成本
npx @cobusgreyling/loop-cost --pattern ci-sweeper
```

## 具体项目预算

### 番茄钟开发阶段预算
| 阶段 | 预计Token消耗 | 时间限制 |
|------|---------------|----------|
| 基础计时功能 | 50K | 30分钟 |
| 按钮控制功能 | 30K | 20分钟 |
| 休息时间逻辑 | 40K | 25分钟 |
| 音效功能 | 20K | 15分钟 |
| 设置界面 | 35K | 20分钟 |
| 测试和修复 | 60K | 40分钟 |

### 总预算上限
- **总Token预算**：500K
- **总时间预算**：3小时
- **最大迭代轮数**：5轮
- **单个功能最大尝试次数**：3次

## 预警机制
- 50%预算消耗：发出警告
- 80%预算消耗：切换为仅报告模式
- 100%预算消耗：立即停止并通知人类
