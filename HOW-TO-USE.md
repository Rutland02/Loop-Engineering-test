# 如何使用自动化任务 - 用户指南

## ?? 概述

您现在已经拥有了完整的自动化报告系统。本指南将帮助您快速上手使用。

## ?? 第一步：理解自动化任务

### 自动化任务做什么？
- ? **每天自动生成项目报告**
- ? **监控PR状态和功能完成度**
- ? **检查预算使用情况**
- ? **不修改任何代码文件**
- ? **不执行构建或测试**
- ? **不自动合并PR**

### 自动化任务不做什么？
- 不会自动修复bug
- 不会自动创建PR
- 不会自动合并代码
- 不会修改项目配置

## ?? 第二步：设置自动化

### 方法A：通过GitHub（推荐）
1. 将项目推送到GitHub仓库
2. 访问仓库的 **"Actions"** 标签页
3. 点击 **"I understand my workflows, go ahead and enable them"**
4. 自动化任务将每天自动运行

### 方法B：本地运行
```bash
# 生成报告
powershell -ExecutionPolicy Bypass -File scripts/generate-report.ps1

# 查看报告
cat reports/pr-status-report.md
```

## ?? 第三步：查看报告

### 报告存储位置
1. **GitHub Actions制品**：在Actions运行详情页面的"Artifacts"部分
2. **本地reports目录**：`reports/pr-status-report.md`
3. **每日报告**：`reports/daily-report.md`（如果配置了）

### 报告内容解读
- **项目状态概览**：当前监控的PR和状态
- **功能完成状态**：哪些功能已完成，哪些待开发
- **测试状态**：测试覆盖情况
- **预算使用情况**：Token消耗和运行次数
- **建议操作**：下一步应该做什么
- **下一步计划**：短期和长期开发计划

## ? 第四步：管理定时任务

### 查看运行历史
1. 访问GitHub仓库的 **"Actions"** 标签页
2. 点击 **"番茄钟项目PR状态报告"** 工作流
3. 查看所有运行记录

### 手动触发
1. 在工作流页面，点击 **"Run workflow"**
2. 选择分支（通常是`main`）
3. 点击 **"Run workflow"**

### 暂停自动化
1. 在工作流页面，点击 **"Disable workflow"**
2. 自动化任务将停止运行
3. 需要时可以重新启用

### 修改运行频率
1. 编辑 `.github/workflows/pr-report.yml` 文件
2. 修改 `cron` 表达式
3. 提交更改

## ?? 第五步：自定义报告

### 修改报告内容
编辑 `scripts/generate-report.ps1` 文件，添加您需要的信息：

```powershell
# 示例：添加自定义字段
$customInfo = @"
## 自定义信息
- 项目开始日期: 2026-07-20
- 预计完成日期: 2026-08-15
- 团队成员: 张三、李四、王五
"@

# 将自定义信息添加到报告中
$report = $report + $customInfo
```

### 添加新的监控指标
在报告脚本中添加新的监控逻辑：

```powershell
# 示例：检查测试覆盖率
$testCoverage = "85%"
$report += "`n## 测试覆盖率`n- 当前覆盖率: $testCoverage`n"
```

## ??? 故障排除

### 问题1：自动化没有运行
**检查步骤：**
1. 访问Actions标签页，查看工作流状态
2. 检查是否有错误信息
3. 验证cron表达式语法
4. 确认工作流已启用

### 问题2：报告内容不完整
**检查步骤：**
1. 查看运行日志中的错误
2. 验证`pr-babysitter-state.md`文件格式
3. 检查PowerShell脚本语法
4. 确认文件路径正确

### 问题3：权限错误
**检查步骤：**
1. 确保GITHUB_TOKEN有足够权限
2. 检查仓库的Actions设置
3. 验证工作流文件权限
4. 查看GitHub文档中的权限说明
